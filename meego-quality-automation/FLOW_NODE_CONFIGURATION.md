# 🎯 飞书项目质量指标流程节点配置方案

## 理解正确了！您要配置的是流程节点

您要在飞书项目的**流程管理**中配置节点，用于跟踪5个质量指标。这不是字段配置，而是工作流节点配置。

## 📊 5个质量指标对应的流程节点

根据您的需求，需要配置以下流程节点来跟踪质量指标：

### 1. 需求Lead Time（需求→上线的时间）
需要配置的节点序列：
- **需求创建** → **方案设计** → **评审** → **开发** → **测试** → **上线**

### 2. 评审一次通过率
关键节点：
- **评审准备**
- **评审中**（记录是否一次通过）
- **评审完成**

### 3. 并行事项吞吐量
监控节点：
- **进行中**（记录并行任务数）
- **已完成**（记录周完成数）

### 4. PRD返工率
版本控制节点：
- **PRD初稿**
- **PRD评审**
- **PRD修改**（记录返工次数）
- **PRD定稿**

### 5. 试点到GA迭代
发布节点：
- **试点启动**
- **试点验证**
- **迭代优化**（记录迭代次数）
- **GA发布**

## 🔧 在飞书项目中配置流程节点

### 当前状态
您已经进入了：
- 路径：空间配置 → 工作项管理 → 需求 → 流程管理
- 看到了"普通任务"流程
- 有一个"开始"节点

### 配置步骤

#### 方法1：修改现有流程（推荐）

1. **点击"普通任务"流程**进入编辑模式

2. **在流程图中添加节点**：
   - 通常通过拖拽或点击连接线来添加节点
   - 或者在节点上悬停，会出现"+"按钮

3. **配置每个节点**：
   ```
   开始 → 需求创建 → 方案设计 → 评审 → 开发 → 测试 → 部署 → 上线 → 结束
   ```

4. **为每个节点设置**：
   - 节点名称（如"需求创建"）
   - 负责人角色
   - 完成条件
   - 时间记录（用于Lead Time计算）

#### 方法2：创建新的质量指标流程

1. **点击"新建流程"**

2. **流程名称**：质量指标追踪流程

3. **添加质量节点序列**：
   ```yaml
   质量指标流程节点:
     1. 需求接收:
        - 记录: 创建时间
        - 负责人: 产品经理

     2. 方案设计:
        - 记录: 开始时间、完成时间
        - 负责人: 技术负责人

     3. 评审节点:
        - 记录: 评审结果（一次通过/需修改）
        - 记录: 评审轮次

     4. 开发节点:
        - 记录: 开始时间、并行任务数
        - 负责人: 开发人员

     5. 测试节点:
        - 记录: 测试通过时间
        - 负责人: 测试人员

     6. 试点部署:
        - 记录: 试点开始时间
        - 记录: 迭代次数

     7. GA发布:
        - 记录: 正式发布时间
        - 计算: 总Lead Time
   ```

## 📝 节点配置详情

### 每个节点需要配置的信息

1. **基础信息配置**
   - 节点名称*
   - 节点ID（系统生成）
   - 显示条件

2. **节点负责人配置**
   - 分配方式
   - 默认人员

3. **节点流转配置**
   - 完成方式（手动/自动）
   - 流转条件
   - 下一节点

4. **数据记录配置**（重要！）
   - 时间戳记录（用于Lead Time计算）
   - 状态记录（用于通过率计算）
   - 数量记录（用于吞吐量计算）

## 🎯 具体配置示例

### 配置"评审"节点（用于评审通过率指标）

1. **节点信息**：
   - 名称：需求评审
   - 类型：审批节点

2. **节点流转**：
   - 完成方式：手动完成
   - 流转条件：评审结果不为空

3. **自定义字段**（在节点中记录）：
   - 评审结果：单选（一次通过/修改后通过/未通过）
   - 评审轮次：数字
   - 评审时间：日期时间

### 配置"开发"节点（用于吞吐量指标）

1. **节点信息**：
   - 名称：开发中
   - 类型：执行节点

2. **节点流转**：
   - 记录并行任务数
   - 记录本周完成数

## 💡 实施建议

1. **先配置核心流程节点**：
   - 需求创建 → 方案 → 评审 → 开发 → 测试 → 上线

2. **在每个节点添加时间记录**：
   - 进入节点时间
   - 离开节点时间

3. **配置节点间的流转规则**：
   - 正常流转路径
   - 返工路径（用于统计返工率）

4. **设置数据收集点**：
   - 在关键节点收集质量数据
   - 自动计算指标

## ⚠️ 注意事项

1. **节点不是字段**：节点是流程中的状态点，字段是数据属性
2. **节点可以包含字段**：在节点配置中可以要求填写特定字段
3. **流程图是核心**：通过流程图定义节点之间的流转关系
4. **数据自动收集**：配置好节点后，系统会自动记录流转时间

## 🚀 下一步操作

1. 在当前界面继续配置节点
2. 通过节点流转设置流程路径
3. 在每个节点配置需要记录的质量数据
4. 保存流程配置
5. 开始使用流程跟踪质量指标

这样配置后，您就可以通过流程节点自动跟踪和计算5个质量指标了！
